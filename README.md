# 🏗️ Meta-Prompt Architect

**把你的模糊想法，自动生成为可以直接用的生产级 Prompt。**

你只需要告诉它你想做什么——它会完成角色建模、认知框架设计、执行逻辑建构、约束条件设定和格式规范，输出一个在任何主流大模型中都能稳定运行的高质量 Prompt。

> 当前版本：**v5.14 Stable Release**
> 适配模型：Claude / GPT-4o / DeepSeek / Qwen

---

## 它解决什么问题

你是否遇到过这些情况：

- 跟 AI 说了半天，回答总是跑偏，没法直接用
- 想写一个好用的 Prompt，但不知道该定义哪些内容
- 网上找来的 Prompt 用起来差点意思，又不知道从哪里改
- 用现成的 Agent 工具，效果不满意却完全没法调整

问题通常不在于你的需求，而在于 Prompt 缺少清晰的角色定义、可验证的约束和明确的执行逻辑。Meta-Prompt Architect 把这些工作自动完成。

## 三分钟看懂它能做什么

**你说：**
> "帮我做一个图片风格逆向提示词工具，我会上传图片，它帮我生成可以复现同款风格的文生图提示词"

**Architect 生成：**

```
# Role: 视觉风格逆向解码专家

## Profile:
- 价值陈述: 专注将任意图像反向解码为高可复现性的文生图提示词
- Prompt Type: 对话型 (Chat)
- Task Category: 图像类 (Image)
- Generated by: Meta-Prompt Architect v5.14

## Cognitive Source:
约翰·伯杰《观看之道》视觉解码方法论 × 文生图提示词工程结构化表达规范
- 伯杰方法论职责：图像视觉符号拆解、构图意图识别、光影色彩关系解读
- 提示词工程规范职责：将视觉分析结论转化为符合 Midjourney / SD / DALL·E 的提示词结构

## Cognitive Framework:
1. 视觉层级优先锚定 — 拆解图像时首先识别视觉权重最高的核心层级；
   当图像为设计海报类复合媒介时，优先分层拆解而非整体分析
2. 风格特征收敛推断 — 若图像呈现多种风格特征，收敛至最显著技法并附加"受……启发"修饰语；
   在风格描述词与具体参数冲突时，优先保留对生成结果影响更大的风格锚点词
3. 不可见参数降级标注 — 无法从图像直接读取的技术参数，一律标注 [推测：基于XX视觉线索]；
   图像信息不足时，追问图像类型声明而非强行输出低置信度提示词

## Workflow:
1. 图像类型声明与分层确认 ...
2. 风格与意图匹配 ...
3. 提示词结构化构建 ...
4. 平台适配与交付 ...

## Constraints: [5条，含双重真实性约束、类型特征约束、占位符上限]

## Initialization:
视觉逆向引擎已就绪。请提供 [图片附件]，并告诉我
[图像类型：摄影作品 / 插画 / 设计海报 / 混合媒介] 以及
[你的目标：逆向生成提示词 / 分析风格 / 迁移到新素材]
```

完整示例见 `examples/` 目录。

## 如何使用

**第一步：** 复制 `Meta-Prompt-Architect-v5.14.md` 的全部内容

**第二步：** 粘贴到任意大模型的对话框（Claude / ChatGPT / DeepSeek / Qwen 均可）

**第三步：** 发送后，Architect 会主动输出引导开场白，告诉你如何开始

**第四步：** 告诉它你的需求，它会自动完成生成

不需要了解任何 Prompt 框架知识，不需要手动填写任何字段。

## 三种使用方式

**方式一：直接描述需求（最简单）**

直接说你想要什么：
- "帮我写一个分析财报的 Prompt"
- "我需要一个小红书文案创作助手"
- "做一个客服知识库问答 Prompt"
- "生成一个图片风格逆向提示词工具"

Architect 会自动判断任务类型并生成完整 Prompt。

**方式二：提供更多背景（获得更高质量输出）**

如果你能补充以下信息，输出质量会显著提升：
- 这个 Prompt 给谁用？（人类用户 / 程序接口 / 自动化流程）
- 主要用在什么平台或产品中？
- 输入是文字、图片，还是两者都有？
- 有没有特别的格式要求或约束条件？

**方式三：提交现有 Prompt 优化**

把你已有的 Prompt 发给 Architect，它会诊断问题并重构为更稳定的版本。这是发现你的 Prompt 哪里出了问题的最快方式。

## v5.14 核心设计

### 两层 Workflow 独立运行

Architect 内部有两套互相独立的 Workflow：

- **Architect Workflow（上层）**：Architect 自身的执行流程，共 5 个阶段，负责分析需求、建模、生成、自检
- **Generated Prompt Workflow（下层）**：Architect 生成的 Prompt 内部的工作流，默认 3-4 步，由契约约束

两者执行层完全独立，互不触发、互不继承执行状态。

### 认知张力机制（v5.11 新增）

这是 v5.14 最重要的设计升级。

对于**判断性、讨论性、创意性任务**（如策略分析、方案评估、深度探讨），Architect 会强制选择两个在核心判断维度上存在**真实分歧**的方法论，而不是两个互补的来源。

判断标准：这两个方法论对同一个问题，能给出不同的结论吗？

如果不能，Architect 会返回重新选择，最多回路 2 次。若仍未通过，会在输出中标注具体分歧场景，供用户手动指定。

这一机制的目的是从认知框架层面引入对抗性，防止 AI 生成只沿着一个方向延伸的"单声道"回答。

对于**执行性任务**（如数据提取、格式转换、代码生成），则选择单一最权威方法论，确保执行精度。

### 契约式约束

每条生成的 Prompt 都满足可验证的输入-输出契约：

- 所有约束必须是布尔式、可量化的硬指令，而非模糊建议
- 生成 Prompt 固定包含 5 条约束：第 1 条为双重真实性触发，末条为占位符上限
- Phase 4 自检通过后才输出，不通过则静默修正

### 场景类型自适应

Architect 会自动判断任务属于哪类场景，并匹配对应的生成规则：

| 场景类型 | 典型用途 | 特征约束 |
|----------|---------|---------|
| 对话型（Chat） | 聊天助手、内容创作 | 追问必须含终止条件 |
| 接口型（Functional） | 数据处理、格式转换 | 线性无分支、格式校验约束 |
| 知识型（RAG） | 知识库问答、文档分析 | 来源约束最高优先级 |
| 代理型（Agentic） | 自动化任务、多步骤执行 | 终止条件、止损约束 |

任务大类（文字类 / 图像类 / 音视频类 / 结构化数据类 / 多模态类）与场景类型组合，共同决定生成策略。

## 为什么用它，而不是直接用封装好的 Agent

用现成的 Agent 工具，你得到的是一个黑盒——效果不满意，但不知道为什么，也没法改。

Architect 生成的 Prompt 是完全透明的：你能看到它在按什么逻辑工作，每条约束在限制什么，Workflow 的每一步在做什么。你可以直接修改任何字段，哪里不对改哪里，不需要猜测。

对于想真正理解和掌控 AI 行为的用户，这个差异至关重要。

## 适用模型

Architect 依赖模型的长上下文理解、多步推理和指令遵从能力，**建议直接使用各平台当前最新的旗舰或次旗舰版本**，模型能力越强，认知张力机制和契约式约束的执行质量越高。

**最低能力基准**，低于此基准可运行但输出质量无法保证：上下文窗口 ≥ 32K tokens，支持多步推理与链式逻辑，指令遵从能力达到主流商用级别。能力不足的常见表现为：认知框架设计流于形式、约束条件无法转化为可验证指令、Phase 4 自检形同虚设。

适配平台包括但不限于 Claude、ChatGPT、DeepSeek、Qwen 等主流平台，无平台锁定，粘贴即用。

## 参与贡献

**提交测试报告（Issue）**

目前最需要社区帮助的是接口型、知识型和代理型的真实使用验证。如果你用 Architect 生成了这三类场景的 Prompt，无论效果好坏，欢迎提交 Issue，告诉我：你的输入是什么、生成的 Prompt 哪里符合预期哪里不符合、实际使用时的效果如何。这类反馈对推进非对话型场景的迭代直接有帮助。

**贡献 Examples（PR）**

如果你用 Architect 生成了效果好的 Prompt，欢迎提交 PR 放入 `examples/` 对应目录。优先欢迎接口型、知识型、代理型的示例——对话型示例相对充足，其他类型的真实案例目前几乎空白。

代理型（Agentic）场景因 ReAct 循环机制的复杂性，v5.14 尚未提供完整模板。如果你在 Agent 开发领域有实践经验，欢迎通过 Issue 参与讨论专项模板的设计方向。
